WHAT IS DEFECT REPORT

  • Its a document to communicate the defects/bugs in the software with developers and other team member
  • Purpose is to communicate the defect as clearly as possible

HOW TO WRITE A GOOD DEFECT REPORT

  • Title:
  • Description:
  • Environment/Platform:
  • Build No:
  • Component:
  • Severity:
  • Steps to reproduce:
  • Expected result:
  • Actual result:
  • Attachments:

Environment: QA

  1. Launch gmail.com on Chrome Browser
  2. Click on "login" link

  4. Click on "login" button

Expected result:
